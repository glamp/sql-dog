What is this?

This is a SQL client that runs entirely in the browser! I built it out of frustration with existing SQL IDEs for OSX. My biggest issue is that most SQL clients are built for database administrators, NOT database USERS. Think about it...who writes the most queries? Not developers but analysts!

What does it do?

  • Run queries in the broswer
  • Predictive type everywhere
  • Saves your queries using git
  • Save results in the browser, or export to Excel/CSV
  • Help with executing long running queries
  • Customizeable macros / snippets
  • Customizeable keyboard shortcuts
  • Query templating

How do I use it?

Some Tips and Tricks

Use ctrl + f! One of the reasons it's great an in-browser editor is that you can use keyboard shortcuts that you're already using on a day-to-day basis. This is the reason we're not using a fancy table display for your query results--just use what you use already!

Try not to execute 20K+ row queries. There's nothing stopping you from doing this, but do you really need to display 20K results? If you just want to export the data, use the `data export` feature.

There are lots of keyboard shortcuts. Use them! You can find a list here. But try and commit `Run` (cmd + enter) to memory.
